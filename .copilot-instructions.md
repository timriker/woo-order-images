# AI Development Instructions for WOO Order Images

## Project Overview
WOO Order Images (WOI) is a WooCommerce plugin for managing customer-uploaded product photos. The plugin handles cropping, rotation, watermarking, and print-sheet generation with order-level consolidation onto 8.5×11 pages.

## Core Principles

### 1. Order-Level, Not Per-Item
- Print sheets are always **order-wide consolidation**, never per-line-item alone
- Print URL: `admin-post.php?action=woi_print_sheet&order_id=<id>` (no `item_id`)
- Rationale: Reduces paper waste, minimizes print jobs, simplifies workflow

### 2. Minimal, Clean HTML
- Avoid noisy `data-*` attributes for runtime-computed geometry
- Inline critical styles only; use CSS classes for reusable rules
- No hidden metadata in DOM; meta stored in WordPress options/post meta

### 3. Customizable Over Hard-Coded
- Print margins → `WOI_Settings` (configurable via admin, not code)
- Watermark size → per-tile formula (`wrap_margin × 24 pt`), not fixed value
- Product specs → product metabox, not inline constants

### 4. Precise Print Production
- **Page bounds**: `height: 11in` + `overflow: hidden` (hard pixel boundary, prevents blank pages)
- **Page breaks**: Only on non-last pages; no space-filler divs between content
- **Watermark bands**: Occupy full bleed strip (not 1/3), font sized to label (1/3 of tab height)
- **Printer margins**: Applied via `@page` CSS rule + `margin-left/right/top/bottom` in rendering

### 5. Image Orientation Auto-Detection
- Compare image aspect ratio to product visible-area aspect ratio
- If mismatch: swap template orientation (width ↔ height in spec)
- Ratio computation: `image_ratio = width / height`; `product_ratio = visible_width / visible_height`

## Code Patterns

### Settings & Configuration
```php
// Always use WOI_Settings static methods for global config
$watermark_text = WOI_Settings::get_watermark_text();
$margin_top     = WOI_Settings::get_print_margin_top();

// Add new option: const + getter + register_setting
const OPTION_NEW_SETTING = 'woi_new_setting';
const DEFAULT_NEW_SETTING = 'default_value';

public static function get_new_setting() {
    return get_option( self::OPTION_NEW_SETTING, self::DEFAULT_NEW_SETTING );
}
```

### Print Page Rendering
- Collect all images from all order items
- Resolve specs (visible area, wrap margin) per item
- Check orientation; auto-swap if needed
- Bin-pack with gap enforcement
- Render absolute-positioned tiles with bleed, visible window, watermark, cut outline

### Watermark Sizing
```php
// Font size = 1/3 of one bleed tab (wrap_margin × 72 pt/in ÷ 3)
$wm_font_pt = max( 4, $spec['wrap_margin'] * 24 );
```

### Tile Packing
```php
$page_width  = 8.5 - $margin_left - $margin_right;
$page_height = 11.0 - $margin_top - $margin_bottom;

// Bin-pack: left-to-right rows, new page on overflow
// Enforce gap (0.125in) between tiles
```

## Common Development Tasks

### Add a New Print Margin Setting
1. Add const + default to `WOI_Settings`
2. Register in `register_settings()`
3. Add render field method (number input, step=0.01, min=0)
4. Add getter method returning float
5. Use in `render_print_page()` to compute page dimensions

### Change Watermark Behavior
- Edit band height/positioning in `render_print_page()`
- Search: `$top_safe`, `$bottom_safe`, `$left_safe`, `$right_safe`
- Bands occupy full safe area; CSS centers content

### Adjust Tile Packing Algorithm
- Modify `flush_row()` and `flush_page()` closures in `render_print_page()`
- Respect `$page_width` and `$page_height` bounds
- Enforce `$gap` (0.125in default) between tiles

### Add Product-Level Setting
1. Add field in `WOI_Admin_Product_Settings::render_metabox()`
2. Save via `update_post_meta()`
3. Retrieve in `get_item_spec()` method
4. Use in `render_print_page()` when rendering tiles

## Performance Considerations

- **Image loading**: Use `getimagesize()` to detect ratio (not full download)
- **Page generation**: Stream output directly; no in-memory buffering of full HTML
- **CSS**: Consolidate into `<style>` tag; minimize external requests during print

## Debugging Tips

- **Print sheet blank pages**: Check `page-break-after:always` on last page (should be conditional)
- **Watermark not rendering**: Check if band height rounds to 0% or if safe area is negative
- **Incorrect tile positioning**: Verify bin-pack X/Y assignment in `flush_row()` logic
- **Margins not applied**: Check `@page` rule in generated `<style>` vs. CSS file

## Git Workflow

- Commit message format: `feat|fix|refactor: [area] description`
  - Example: `feat: print add per-tile watermark sizing`
  - Example: `fix: admin button appears on order-level actions`
- Keep commits atomic and logically grouped
